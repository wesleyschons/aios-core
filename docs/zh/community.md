# Synkra AIOX 社区

> 🇧🇷 [葡萄牙語版本](../COMMUNITY-PT.md)

欢迎来到 Synkra AIOX 社区！

我们正在一起构建 AI 编排开发的未来。

## 我们的价值观

- **合作胜于竞争** - 我们一起成长
- **包容性** - 欢迎所有人，无论经验水平如何
- **透明度** - 开放讨论，开放决策
- **质量** - 我们关心把事情做好

## 入门指南

### 第一步

1. 给仓库加星标
2. 阅读 [README](../README.md)
3. 设置您的 [开发环境](../CONTRIBUTING.md#getting-started)
4. 在 [讨论](https://github.com/SynkraAI/aiox-core/discussions) 中介绍自己

### 寻找您的第一个贡献

- 查找标记为 [`good-first-issue`](https://github.com/SynkraAI/aiox-core/labels/good-first-issue) 的 issue
- 检查 [`help-wanted`](https://github.com/SynkraAI/aiox-core/labels/help-wanted) 以获得更复杂的任务
- 浏览 [开放讨论](https://github.com/SynkraAI/aiox-core/discussions) 以帮助他人

## 沟通渠道

### GitHub 讨论（主要）

我们所有 AIOX 仓库的主要沟通中心：

- **公告** - 来自维护者的项目更新
- **常规** - 关于 AIOX 的一般讨论
- **想法** - 提议新功能和改进
- **问答** - 获得技术问题的帮助
- **展示和讲述** - 分享您使用 AIOX 创建的项目
- **故障排除** - 获得解决问题的帮助
- **小队** - 关于 AIOX Squads（模块化代理团队）的讨论
- **MCP 生态系统** - 关于 MCP 工具和集成的讨论

[加入讨论](https://github.com/SynkraAI/aiox-core/discussions)

### Issue 跟踪器

用于错误报告和功能请求：

- [aiox-core Issues](https://github.com/SynkraAI/aiox-core/issues) - 核心框架
- [aiox-squads Issues](https://github.com/SynkraAI/aiox-squads/issues) - AIOX Squads
- [mcp-ecosystem Issues](https://github.com/SynkraAI/mcp-ecosystem/issues) - MCP 工具

## 如何贡献

### 代码贡献

1. Fork 仓库
2. 创建功能分支（`git checkout -b feature/amazing-feature`）
3. 进行更改
4. 运行测试（`npm test`）
5. 提交 Pull Request

详见 [CONTRIBUTING.md](../CONTRIBUTING.md)。

### 非代码贡献

我们重视所有类型的贡献：

- **文档** - 修复拼写错误，改进说明
- **翻译** - 帮助翻译文档
- **错误报告** - 报告您发现的问题
- **想法** - 分享您对改进的想法
- **设计** - UI/UX 建议
- **倡导** - 博客文章、讲演、教程

### Squads

创建并分享您自己的 Squads（模块化 AI 代理团队）！

Squads 是在特定领域协作工作的专业 AI 代理团队：
- **ETL Squad** - 数据收集和转换
- **Creator Squad** - 内容生成

详见 [docs/Squads.md](../docs/Squads.md)。

## 社区角色

### 贡献者

任何以任何方式对 AIOX 做出贡献的人。
- 在我们的 [贡献者页面](https://github.com/SynkraAI/aiox-core/graphs/contributors) 上列出
- 在发布说明中提及重大贡献

### 维护者

审查 PR 并指导项目的核心团队成员：

- [@SynkraAI](https://github.com/SynkraAI) - 项目负责人

### 成为维护者

活跃的贡献者可能被邀请成为维护者。我们寻找：
- 持续的高质量贡献
- 有益的社区互动
- 对项目目标的理解

## 认可

### 贡献者墙

所有贡献者都在我们的 [贡献者页面](https://github.com/SynkraAI/aiox-core/graphs/contributors) 上被认可。

### 发布致谢

重大贡献在发布说明中被认可。

## 治理

### 决策制定

- **小决定**：维护者可以决定
- **重大决定**：在 GitHub 讨论中讨论
- **破坏性更改**：需要 RFC 流程

### RFC 流程

对于重大更改：

1. 使用 `[RFC]` 前缀打开讨论
2. 社区提供反馈
3. 维护者做出最终决定
4. 记录决定

### 行为准则

我们遵循 [贡献者契约](../CODE_OF_CONDUCT.md)。请阅读并尊重它。

报告违反行为至：conduct@SynkraAI.com

## 获得帮助

### 被某件事卡住了？

1. 查看 [文档](../)
2. 搜索 [现有讨论](https://github.com/SynkraAI/aiox-core/discussions)
3. 在问答讨论中提问
4. 打开故障排除讨论

### 发现了一个 bug？

1. 搜索 [现有 issue](https://github.com/SynkraAI/aiox-core/issues)
2. 如果是新的，[打开错误报告](https://github.com/SynkraAI/aiox-core/issues/new?template=bug_report.md)

### 有想法吗？

1. 在 [想法](https://github.com/SynkraAI/aiox-core/discussions/categories/ideas) 中检查是否存在
2. 如果是新的，[分享您的想法](https://github.com/SynkraAI/aiox-core/discussions/new?category=ideas)
3. 阅读我们的 [功能请求流程](../docs/FEATURE_PROCESS.md) 了解详细指南

## 项目路线图

想知道 AIOX 的发展方向吗？查看我们的公开路线图：

- [ROADMAP.md](../ROADMAP.md) - 高层愿景和计划功能
- [GitHub 项目](https://github.com/orgs/SynkraAI/projects/1) - 详细跟踪板

路线图每月更新并反映社区意见。您可以通过以下方式影响我们的方向：

1. **投票**支持 [讨论](https://github.com/SynkraAI/aiox-core/discussions/categories/ideas) 中的想法
2. **通过 [RFC 流程](/../.github/RFC_TEMPLATE.md) 提议新功能**
3. **直接贡献**到计划功能

> 路线图项目是计划，不是承诺。优先级可能会根据社区需求和技术限制而改变。

### 功能请求流程

我们有一个结构化的流程来提议新功能：

1. **快速想法** - 在"想法"类别中打开讨论
2. **RFC 流程** - 对于重大功能，使用我们的 [模板](/../.github/RFC_TEMPLATE.md) 编写 RFC
3. **社区投票** - 使用 :+1: 反应表示支持
4. **实施** - 批准的想法进入我们的待办事项

详见 [功能请求流程](../docs/FEATURE_PROCESS.md)。
